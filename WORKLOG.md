## 2026-06-12

### 今日目標（下午）

提升 Three.js 場景的「實體感」：在不離開 Web 技術棧、不犧牲互動功能的前提下，讓 3D 渲染更接近建築示意渲染的質感（取代 UE5 後製路線的第一步）。

### 完成事項（下午）

- 渲染管線升級為 PBR 工作流：啟用 ACES Filmic tone mapping，整體色調從「平光塑膠感」轉為攝影感。
- 加入 IBL 環境光照：以 `RoomEnvironment` 產生環境貼圖（`PMREMGenerator`），所有 `MeshStandardMaterial` 物件（建築、屋頂、設備機殼）獲得真實的環境反射與柔和補光。
- 啟用即時陰影：太陽方向光改為投射 PCF 柔和陰影（2048 陰影貼圖、涵蓋全校區），建築投影至地面、操場與平面圖上；半透明（X-ray / 聚焦淡出）狀態的建築自動停止投影避免假影。
- 加入 N8AO 螢幕空間環境光遮蔽（half-res，行動裝置可負擔）：建築底部、樓棟之間、屋簷交接處出現自然的接觸陰影，體積感明顯提升。
- 後製管線改用 `EffectComposer`（4x MSAA render target + N8AO + OutputPass），維持原本的截圖功能與 Raycaster 互動不變。
- 重新平衡光照預算（半球光 0.4 / 太陽光 1.25 / 環境光 0.3）：日照面與陰影面有明確明暗差，平面圖底圖與狀態色（紅 / 黃 / 綠）維持可辨識，不過曝。
- 平面圖底圖、操場 / 球場 / 跑道貼面改為受光材質並接收陰影，建築陰影能自然落在底圖上。
- 立面窗格改為玻璃質感材質（低粗糙度 + 金屬感 + 環境反射），窗框改為受光材質。
- 霧距離由 115–210 調整為 150–290，俯瞰全校時建築不再被霧洗白。
- 驗證：`npm run build` 通過、`verify-visual` 桌面 / 行動雙視口通過、無 console 錯誤；設備狀態 / 訊號熱區 / 實體線路 / 樓層規劃模式均正常，與線上舊版截圖對比確認立體感提升。
