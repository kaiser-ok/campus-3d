# 工作日誌

## 2026-05-18

### 今日目標

改善匯入不同學校平面圖後的修正流程，讓使用者可以刪除判讀不理想的學校資料，重新匯入前先檢查建築位置是否合理。

### 完成事項

- 新增目前學校管理區塊，選到匯入學校時可直接執行「編輯」、「重新匯入」或「刪除學校」。
- 新增目前學校編輯器，可修改學校名稱、更新 JPG / PNG 底圖，並調整每棟建築的名稱、樓層、地下樓層、顏色、X/Z 位置、W/D 尺寸與房間文字。
- 編輯器支援新增或移除建築，儲存後會同步更新 localStorage 與目前 3D 場景。
- 刪除匯入學校時加入確認視窗，並會同步移除 localStorage 中的學校資料與底圖。
- 若刪除的是目前正在看的學校，場景會自動回到預設的壽山高中。
- 匯入精靈第二步的建築框現在可直接拖曳移動，也可拉白色控制點縮放，方便在套用 3D 前修正 AI 判讀位置。
- 第二步新增「自動收斂框線」，會依底圖中的線條、文字與色塊內容，把疑似過大的 AI 框縮回建築本體。
- 匯入精靈最後一步改為「預覽套用」，在套用前以底圖疊加建築框，讓使用者先確認建築物位置。
- 預覽畫面加入異常提示，會標示疑似過大、過長或互相重疊的建築框。
- 自動分析結果會先排除常見非建築物名稱，並對過度重疊的建築框做幾何整理，降低像跑道、球場或交錯量體被誤當成建築的機率。
- AI prompt 補強：若建築只有一排教室，避免把下方空白區、中庭或廣場一起框入，例如教學大樓 A 區。
- 新增實體線路資料模型 `networkLinks`，可保存每台 AP / switch 對應的 switch、port、patch panel、patch port、VLAN、線號、媒介、光纖芯數、上行設備與狀態。
- 右側控制面板新增「實體線路資料」卡片，可匯入 CSV / JSON 線路對照表。
- 線路匯入支援同步新增或更新 AP / switch，欄位可包含 `deviceId`、`type`、`name`、`building`、`floor`、`x`、`z`、`switchId`、`switchPort`、`patchPanel`、`patchPort`、`vlan`、`cableId`、`medium`、`fiberCore`、`uplinkTo`、`status`、`note`。
- 點選 AP / switch 時，右側細節面板會顯示實體線路路徑、switch port、patch panel、VLAN、線號、光纖芯與線路狀態。
- 3D 線路圖會依匯入資料區分 Cat6、光纖與異常線路；選取設備時線路標籤會顯示對應 port。
- 修正 Lucide `Map` 圖示與 JavaScript 內建 `Map()` 撞名造成頁面載入失敗的問題。
- 新增「載入範例」線路按鈕，讓目前瀏覽器可直接看到範例 IDF、故障 AP 與高流量 AP 的實體線路效果，不再只存在自動化測試瀏覽器。

### 驗證紀錄

- `npm run build`：通過。
- `node scripts/verify-visual.mjs`：通過。
- Playwright CSV 匯入測試：匯入臨時 AP / switch 線路資料後，可新增設備、切換到實體線路模式，並在設備細節看到 `Gi1/0/24` 與對應 patch panel。
- Playwright 範例線路測試：按下「載入範例」後，可看到範例故障 AP、`Gi1/0/48`、`DEMO-CAT6-048` 與紅色線路狀態。
- 目前學校編輯器流程測試：可開啟編輯器、修改學校名稱、修改建築名稱與尺寸並儲存。
- 底圖更新流程測試：可上傳 JPG / PNG 並寫入該學校的 localStorage 底圖資料。
- 驗證內容包含桌面與手機 viewport、WebGL canvas 非空白、工具列可見、模式列可見。

### 目前限制

- AI 自動判讀仍可能受平面圖標註、陰影、格線密度與掃描品質影響。
- 拖曳校正目前是矩形建築框校正，可移動與縮放，但尚未加入旋轉、多邊形或曲線輪廓。
- 實體線路已可匯入設備對照資料，但 3D 走線仍以建築外側線槽與垂直管道推估，尚未做到逐段 cable tray、管道轉彎、弱電井位置與 patch panel 櫃位的精準建模。
- 匯入資料目前是 CSV / JSON 手動檔案，尚未直接串接 UniFi、switch SNMP/API、NMS、竣工圖或資產系統。
- 第一教學大樓等位置若 AI 誤判，建議先刪除該學校後重新匯入，並在第二步拖曳校正後再套用。

### 下一階段建議

- 加入建築框旋轉與多邊形輪廓校正，處理斜角建築。
- 加入「忽略此建築」與「拆成兩棟」操作，處理 L 型、長廊型或複合建築。
- 保留 AI 原始判讀與使用者校正版本，方便比較與回復。
- 建立 patch panel / switch port 編輯器，讓使用者不用直接改 CSV 也能維護實體線路。
- 加入 VLAN、光纖芯數、線路異常、port 使用率的篩選與高亮。
- 串接 UniFi / switch / NMS API，將 AP 狀態與 port uplink 狀態自動同步到 3D 場景。

- 選中建築物後，屋頂會顯示資訊平台，用來呈現目前樓層、房間數、該樓層設備數與故障數。

## 2026-05-16

### 專案

壽山高中 WiFi 3D 校園監控圖

### 今日目標

將學校 2D 平面圖轉成可在 Web 中互動檢視的 3D 校園監控介面，並可顯示 AP、switch、設備狀態、WiFi 訊號問題區與流量熱點。

### 完成事項

- 建立 Vite + React 專案，使用 Three.js 製作可旋轉、縮放、俯視與透視檢視的 3D 場景。
- 將原始學校平面圖放入 3D 地面底圖，並依照圖面位置建立主要建築量體。
- 加入監控模式切換：
  - 設備狀態
  - 訊號熱區
  - 用戶流量
  - 樓層規劃
- 加入 AP / switch 示範資料，顯示設備狀態、樓層、用戶數、流量與頻道資訊。
- 設備狀態以顏色區分：
  - 淡綠色：正常
  - 黃色：警告
  - 紅色：故障
- 加入 WiFi 熱區視覺化，標示訊號差、斷線、流量高與壅塞區域。
- 建築物加入樓層分隔線、樓層標籤與簡單窗帶，不再只是 solid box。
- 加入教室/空間編號資料，點選建築物時會在 3D 立面與右側資訊面板顯示房號。
- 點選建築物或設備時，鏡頭會自動聚焦放大到該目標。
- AP / switch 改為依樓層高度顯示，而不是全部放在屋頂。
- 故障設備會在對應樓層位置顯示紅色標記、紅色警示圈與故障標籤。
- AP / switch 圖層開啟時，建築物會自動進入 X-ray 半透明模式，避免設備被建築遮住。
- 建立 Playwright 視覺驗證腳本，檢查桌面與手機尺寸下 canvas 是否正確渲染、工具列是否在畫面內。
- 初始化 git repository 並建立第一個 commit。
